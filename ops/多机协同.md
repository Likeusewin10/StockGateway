# 多机 / 多 Agent 协同 — Git 地基

> 本阶段定位:**Git + CLAUDE.md + handoff 文件**,不做 SSH / Agent-to-Agent RPC。
> 本地 Agent `git pull` 读 `CLAUDE.md` 即懂服务器;**指挥服务器侧 Agent 有两条路**:
> ① 异步实时指挥走 **MCP `agent_*` 工具**（已落地,见第 0 节）;② 离线/留痕任务走 `ops/handoff/`。

## 0. 经 MCP 调服务器 Agent（实时指挥,已落地）

本地 Agent 通过 8765 网关的三个工具,在服务器上**异步**起一次性 `claude`/`codex` 子进程干活:

```
agent_agent_run(engine="claude", prompt="<指令>")  → {task_id, status:"running"}
agent_agent_status(task_id)  → 轮询至 done/failed/timeout
agent_agent_result(task_id)  → 取 {output, error, returncode}
```

- **鉴权**:复用网关 X-API-Key(无第二层),由 `ApiKeyAuthMiddleware` 统一覆盖。
- **执行边界**:cwd 锁死 StockSDK 项目目录;engine 限 `claude`/`codex` 白名单。
- **上下文**:服务器 `CLAUDE.md` 每次随 prompt 自动加载(第一步不做常驻会话/跨任务上下文)。
- **🔴 安全**:子进程全自动非交互(`--dangerously-*`),拿到 X-API-Key 即可在服务器执行任意命令。
  API_KEY 必须强随机且只发可信机器。详见 [`docs/MCP中转网关.md`](../docs/MCP中转网关.md) 第六节。

> 与 handoff 的分工:**实时、要立刻拿结果** → MCP `agent_*`;**离线、要留痕/可追溯** → `ops/handoff/`。
> 两者都不替代 Git——代码改动仍走 worktree + 分支 + push(下面各节)。

## 1. worktree:服务器侧 Agent 的独立工作区

服务器侧 Agent 改代码时,用 `git worktree` 开独立目录 + 独立分支,避免与主工作区（跑着服务的那份）互相干扰:

```bash
# 在仓库根:为某个任务开一个 worktree（独立分支 agent/<任务>）
git worktree add ../StockSDK-agent-task1 -b agent/task1

# 在该目录里改代码、跑测试,与主工作区互不影响
cd ../StockSDK-agent-task1
.venv-api/Scripts/python -m pytest --cov=stocksdk    # venv 在主目录;按需在新目录建或复用

# 完成、合并后清理
git worktree remove ../StockSDK-agent-task1
```

> 主工作区（运行服务的那份）始终留在稳定分支;Agent 的实验都在 worktree 分支里,验证通过再合。

## 2. Git 中心 remote（尚未选定，确定后一行接入）

当前仓库**无任何 remote**,多机协同前必须先有一个中心。两条路（本轮不拍板）:
- **自建**（代码不出本地网络,符合凭据只留本机）:办公室服务器上 `git init --bare` 或装轻量 Gitea,各机 push/pull。
- **私有 GitHub/GitLab**:省运维,但代码上第三方云——**push 前务必确认仓库零密钥**。

确定后接入:
```bash
git remote add origin <地址>
git push -u origin master
# 其他机器:git clone <地址> 然后 git pull 即可读 CLAUDE.md / ops/board / ops/handoff
```

## 3. 🔴 入库 / push 前密钥检查清单

每次 push 到中心 remote 前逐条确认:

- [ ] `git status` 里**无** `.env` / `userInfo`（应被 gitignore;若出现说明 gitignore 失效）
- [ ] `git diff --cached` 里**无** token / 账号 / 密码 / 完整内网 URL / JWT
- [ ] 无大文件（`THSDataInterface_*.zip` 等已 gitignore;`git status` 不应出现）
- [ ] 新加的文档里示例凭据用占位符（`<key>` / `<secret>`),非真实值
- [ ] 快速扫一遍:`git diff --cached | grep -iE "password|token|secret|jwt|THS_PWD|API_KEY=" ` 应只命中占位/变量名

> gitignore 已覆盖:`.env`、`userInfo`、`*.log`、`.venv*`、`downloads/`、`THSDataInterface_*.zip`、
> `THSDataInterface/`、`.obsidian/`、`.coverage`。

## 4. 协同文件分工

| 路径 | 谁写 | 谁看 | 作用 |
|---|---|---|---|
| `CLAUDE.md` | 维护者 | 所有 Agent | 服务器全貌,新 Agent 必读 |
| `ops/handoff/inbox/` | 派发 Agent | 执行 Agent | 一次性任务派发 |
| `ops/handoff/done/` | 执行 Agent | 派发 Agent | 任务结论回写 |
| `ops/board/` | 任意 Agent | 人 + Agent | 进度/需求/里程碑 |
| `docs/运维手册.md` | 维护者 | 运维 Agent | 启停/探活/故障处置 |
