# 取数纪律 · Agent 提示词文档

> 面向所有取数 Agent(Coco / Abby / Dex 等)与人工。**最高优先级规则**:违反将导致数据源账号被厂商停用。
> 本文分两部分:①【可直接粘贴的系统提示词块】——塞进 Agent 系统提示即可;②【背景与依据】——给人看的来龙去脉。

## 〇、致接收本文档的 Agent(收到后立即执行)

你(那侧的取数 Agent,如 Coco)收到本文档后,**须把下方【一、系统提示词块】原样写入你自己的 rules**,让它对你后续所有取数长期生效:

1. 在你的工作区**新建/更新你侧的规则文件**——优先 `.claude/rules/data-fetch-discipline.md`;若你侧用 `CLAUDE.md` 数据铁律节,则并入该节并指回本文档。
2. **照抄下方规则块,不要改写、不要精简**——这是封号红线,任何删改都可能漏掉铁律。
3. 写入后**回报一句「取数纪律已写入 <文件路径>,已生效」**,并在之后每次取数前自查是否符合五条铁律。
4. 若你侧已有本纪律,则**比对版本**:以本文档为准,发现差异就更新,不降级。

> 为什么让你自己写:纪律须落在你能自动加载的 rules 里才对你生效;但**内容以本文档为唯一正典**,你只做"复制落地",不做"自行拟定"。

## 一、系统提示词块(复制此段进 Agent 系统提示词 / 你侧 rules)

```
# 取数纪律(最高优先级 · 违反将导致数据源账号被封停)

## 核心原则:目标驱动取数,禁止发现式遍历
每一次取数都必须服务于一个明确的业务需求,而不是"为了知道有什么"。
封号的根源不是取数量大,而是【无目标地把指标/代码字典从头舐到尾(发现式遍历/探测)】。

## 五条铁律(任一违反即停并上报)
1. 【禁止遍历字典】绝不在线遍历任何厂商的"指标树 / 字段字典 / 数据浏览器"接口
   来发现有哪些指标。要知道有哪些字段,只查【本地静态字段手册】,不联网枚举:
   - iFinD: docs/catalog/ths/*_indicators.csv、docs/catalog/同花顺iFinD指标字段手册.md
   - EM:    docs/catalog/em/*.csv、docs/catalog/东方财富EM指标字段手册.md
   - Wind:  docs/catalog/WindPy字段手册.md(可直接喂 wsd/wss 的口径)
   手册没有的字段:先补手册 / 先问,不要拿接口去试。

2. 【需要什么取什么】每次请求必须是明确三元组:(证券代码集 + 指标集 + 时间范围)。
   三者都要有实际业务来由;不允许任一维用"全部 / 通配"兜底。

3. 【禁止发现式探测】不得为"试出哪个字段/代码有效"而批量试探接口
   (例:把候选码逐个喂取数函数看哪个不报错)。字段有效性一律查手册,不靠接口反馈。

4. 【禁止全量笛卡尔积】不得对"全市场标的 × 全部指标"做拉取。
   按真实需要的股票池 + 指标集,分批小拉;宁可多轮小批,不做一次性全扫。

5. 【批量优先于循环】同一批需求,用一次请求携带多 codes / 多 indicators,
   不要一个代码一个请求、一个指标一个请求地循环轰炸。

## 限速与配额自律
6. 控制并发与频率:默认串行或低并发,请求间留间隔,避免短时间高频。
7. 主动查剩余额度(iFinD 用 THS_DataStatistics),接近上限即降速 / 暂停。
8. 会话约束:SDK 单点登录,勿多机抢登同一账号;HTTP access_token 有 IP 上限(iFinD 20 IP/token)。

## 遇风控信号立即刹车(禁止重试轰炸)
9. 报错含"额度 / 流量 / 频率 / Device exceed limit / 被限"等 → 立刻停止该源取数,
   记录并上报;不要自动重试,更不要换参数继续打。

## 数据源选择叠加"封号风险"维
10. 排数据源优先级时,除数据质量外必须叠加【遍历友好度 / 封号风险】:
    - 明确目标的批量取数 → 走高质量源(iFinD 优先)。
    - "有哪些字段 / 代码"类需求 → 永远走本地静态手册,从取数源彻底剥离。
```

## 二、背景与依据(给人看)

### 为什么有这份纪律
2026-07 iFinD 数据接口账号被厂商停用。客户经理明确反馈根因:**大量访问数据浏览器的指标、"一个个去遍历"**;要求「需要什么指标就访问什么指标,不能遍历」。

### 犯人:字段字典遍历,不是正常取数
- ❌ **封号行为(遍历)**:`scripts/catalog/ifind_crawl.js` —— 从 `seq=0` 起递归爬 `data_browser_web/index_tree/list_by_seq` 的**全品种指标树**,收 22940 指标,并发不停打。这就是「数据浏览器遍历」。
- ❌ **同类反模式(别源)**:Wind 的 `scripts/catalog/wind_probe_fields.py` / `wind_wss_calib*.py` —— 拿 28327 个候选字段码暴力喂 `wss` 二分找有效字段。换到 iFinD 上必被封。
- ✅ **正常取数(不封)**:运行时服务 `stocksdk/routes_ths.py` 的 `THS_HistoryQuotes` / `THS_BasicData` 等 —— 指定 code + 指标 + 时间,「需要什么取什么」。

### 关键区分
封号看的**不是取数总量**,而是**行为形态**:
- 「明确目标、批量、指定 code+指标」= 安全。
- 「无目标、从头到尾舐字典、试探有效性」= 滥用,被判违规。

运行时取数本身安全;**真正风险在于 AI Agent 通过透传 `/ths/call/*` 去循环枚举指标**——那等于在生产账号上复现了爬虫行为。故本纪律必须写进每个取数 Agent 的系统提示词。

### 字段字典去哪查(不联网枚举的替代)
全量字段字典**已一次性落地在本地**,直接查静态文件即可,无需再碰厂商接口:

| 源 | 本地手册 | 说明 |
|---|---|---|
| iFinD | `docs/catalog/ths/*_indicators.csv`、`docs/catalog/同花顺iFinD指标字段手册.md` | 22940 指标 / 13 品种 |
| EM | `docs/catalog/em/*.csv`、`docs/catalog/东方财富EM指标字段手册.md` | 18512 字段 |
| Wind | `docs/catalog/WindPy字段手册.md`(`windpy_fields.csv`) | 5359 个可直接喂 `wsd/wss` 的口径 |

> ⚠ Wind 另有 `Wind指标字段手册.md`(xla 并集 10069)仅用于**查中文名/找指标**,不能直接喂 `wss`。

### 对数据源优先级评估的影响
在既有「iFinD > Choice > Tushare > AKShare > 网页端」质量优先级之上,叠加**封号风险维**:
- **取数类需求**(明确 code+指标):按质量优先,iFinD 优先。
- **字段发现类需求**(有哪些指标/代码):**从取数源彻底剥离**,一律查本地手册。
