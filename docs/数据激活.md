数据源优先级重排序激活手册
**本文给 YC 重登服务器并恢复 iFinD + Choice 后使用。目标是在不改业务逻辑的前提下, 把统一 fetch_service / SourcePriorityResolver 的字段级取数链切到付费源主导:
iFind > Choice > Tushare > AKShare > 网页端

本文是运行手册,不改 src/。真实密钥、网关地址和账号只放服务器本机环境变量或本机 secret 文件,不要写进仓库。
当前仓库状态
已具备:
iFinD HTTP 路径: ifind-token-check, ifind-quantapi-check,
ifind-realtime-check, ifind-import, ifind-refresh,
ifind-refresh-loop。
Tushare MCP 路径: tushare-mcp-check, tushare-import,
tushare-moneyflow, tushare-holder-change, tushare-report-calendar,
tushare-market-pool-rank。
通用 HTTP/JSON 最新价路径: market-data-http-refresh 和
market-data-http-refresh-loop,可临时承接 Choice 或券商行情网关。
fetch service 目标链路已在 docs/research/realtime-fetch-impl.md 设计:
业务调用 fetch_service(field, symbols, ...),内部由 resolver 按字段链路短路命中。
仍需注意:
仓库当前没有 Choice 专用 provider/CLI。Choice 恢复后,先用本机
sync_choice_* 或通用 HTTP/JSON 网关验证;后续再补 choice-check、
choice-import、choice-refresh adapter。
ETF 申购净流入当前仍是待接入项。不能用 ETF 二级市场成交额或主力净流入冒充
一级市场申购净流入。
Tushare daily_basic、margin_detail、fund_portfolio、index_weight
当前主要通过 tushare-market-pool-rank 使用,还不是通用字段 resolver 的直接入口。
激活原则
先验证高权重源健康,再打开字段开关。
单字段命中 ok 即停止,不得同时让 AKShare/网页端覆盖付费源结果。
高权重源认证失败、限流、网络不可用时,记录 attempt 后降级;UI/API 必须展示实际命中
source/status/as_of/fetched_at。
强实时字段不能用日频数据伪装。latest_quote、intraday_bar 只能在 iFinD/Choice
实时源命中时标 ok;Tushare 日频只能是 stale 或 partial。
切换只改服务器本机配置和环境变量,不要在仓库中写 token、URL、账号。

网关可用性检查脚本
3.1 iFinD 必跑检查
只验证 token 交换:
trading-system ifind-token-check

验证日 K/OHLCV/换手率历史接口:
trading-system ifind-quantapi-check \
  --symbol 600519 \
  --start 2026-06-01T00:00:00+08:00 \
  --end 2026-06-23T00:00:00+08:00 \
  --fields open,high,low,close,pre_close,volume,turnover_rate

验证实时价和盘中换手率:
trading-system ifind-realtime-check \
  --symbol 600519 \
  --fields latest,preClose,open,high,low,volume,turnoverRate

通过标准:
IFIND_TOKEN_CHECK 输出 access_token_received=true。
IFIND_QUANTAPI_CHECK 至少一个 check status=ok rows>0。
IFIND_REALTIME_CHECK 输出 quote_available=true,且 price、timestamp 非空。
输出不能出现 token、完整 URL、账号。
3.2 Choice 检查
仓库当前没有 Choice 专用 CLI。服务器恢复 Choice 后按实际接入形态二选一:

rows=<positive integer>

B. 若 Choice 先以 HTTP/JSON quote 网关接入
先用通用 quote 刷新命令做只读 smoke。该命令会写入本地行情缓存,但不会下单:

字段级配置开关
推荐把 resolver 配置放在服务器本机文件,例如 ~/.sts-source-priority.toml。 上线时由运行进程读取;仓库内只保留模板或文档。
4.1 全局 source 开关
[source_priority]
rank = ["ifind", "choice", "tushare", "akshare", "web"]
record_attempts = true
stop_on_status = ["ok"]

[source_priority.sources.ifind]
enabled = true
refresh_token_env = "IFIND_REFRESH_TOKEN"
healthcheck = "trading-system ifind-token-check && trading-system ifind-realtime-check --symbol 600519 --fields latest,preClose,open,high,low,volume,turnoverRate"

[source_priority.sources.choice]
enabled = true
token_env = "CHOICE_TOKEN"
healthcheck = "choice_realtime_check.sh 600519"

[source_priority.sources.tushare]
enabled = true
server_url_env = "TUSHARE_MCP_URL"
healthcheck = "trading-system tushare-mcp-check --symbol 600519"

[source_priority.sources.akshare]
enabled = true

[source_priority.sources.web]
enabled = false
allow_only_as_last_resort = true

规则:
rank 只能禁用源,不能改变相对顺序。
choice.enabled=true 只有在 Choice 检查脚本通过后才能打开。
web.enabled=false 是默认生产设置;只有排障或低优先级补洞时短期开启。

ETF 申购净流入接入步骤
5.1 口径
基金级:
etf_net_creation_units = etf_shares_today - etf_shares_previous_trade_day
etf_net_creation_value = etf_net_creation_units * nav_or_close_price

个股级:
stock_etf_net_inflow = sum(etf_net_creation_value * stock_weight_in_etf)

stock_weight_in_etf 优先取当日 PCF/申购赎回清单。取不到 PCF 时,可退化为指数权重 或最近一期基金持仓权重,但必须标 status=partial。
5.2 iFinD / Choice 优先接入
YC 重登后,先在服务器上问清并验证这些字段:
ETF 当日份额或申赎份额;
ETF NAV、收盘价或 IOPV,明确单位;
ETF PCF/申购赎回清单中的成分证券代码、数量、现金替代、权重;
若厂商提供个股级 ETF 申购净流入聚合值,确认方向、单位、是否已按成分权重分摊。
检查脚本建议:
ifind_etf_creation_check.sh 510300 2026-06-23
ifind_etf_pcf_check.sh 510300 2026-06-23
choice_etf_creation_check.sh 510300 2026-06-23
choice_etf_pcf_check.sh 510300 2026-06-23

通过标准:
同一 ETF 在 T 与 T-1 都能取到份额或申赎份额。
能取到 NAV/收盘价/IOPV 中至少一个可复现金额的价格。
个股级 ok 必须有当日 PCF 或厂商明确的个股级分摊字段。
只有基金级份额变化、没有个股权重时,只能返回基金级结果;个股字段
status=unavailable 或 partial,不得写 ok。
5.3 Tushare / 免费源备选
Tushare 当前可辅助:
fund_portfolio:最近一期基金持仓;
index_weight:指数权重;
daily_basic:市值归一;
moneyflow:个股资金交叉验证。
这些不能等价于当日 ETF PCF。若用 Tushare 估算:
source = "tushare:fund_portfolio_index_weight_estimate"
status = "partial"
notes includes "ETF flow allocated by stale holdings/index weight, not daily PCF"

AKShare/网页端 ETF 份额、规模、PCF 只能作为最低优先级估算或人工核对。

7.2 降级验证
临时禁用 iFinD token 或把 iFinD source 标 disabled,确认:
resolver 记录 iFinD attempt 为 authentication、network 或 skipped。
Choice 成为第一命中源。
Choice 也失败时才落到 Tushare/AKShare。
恢复 iFinD 后,下一轮自动回到 iFinD。
7.3 数据质量验证
核对:
日 K 最近 5 个交易日 OHLCV 与 iFinD/Choice 终端一致。
最新价时间戳在盘中不超过 normal_max_age_seconds=300;强实时不超过 30 秒。
换手率单位一致,统一为百分比数值,如 5.6 表示 5.6%。
市值统一为人民币元;Tushare daily_basic.total_mv/circ_mv 从 10k CNY 转成 CNY。
资金流金额统一为人民币元;Tushare moneyflow.net_mf_amount 从 10k CNY 转成 CNY。
概念/行业成员命中时保存 source 和 as_of,不要只保存板块名称。
7.4 安全验证
确认:
所有检查脚本只读,不会调用 QMT/PTrade 下单接口。
refresh 可以评估告警、生成待审草案、评估交易计划,但不得审批信号。
live 仍保持 LIVE_TRADING_ENABLED=false,真实交易仍需显式 confirmation token。
