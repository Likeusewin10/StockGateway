> 🔴 **Wind 与前两家本质不同**：EM/iFinD 都有独立公网命令生成器把全量指标树暴露在后台 JSON 接口里（登录浏览器即可爬）；**Wind 没有**——其全量字段字典被加密锁在本地终端内，所有可编程旁路均已实测堵死（公网帮助中心 `wx.wind.com.cn` 只放函数手册、本机终端不开 TCP 端口、SPA bundle 不含树、CDP 调试端口被屏蔽、`Indicator.xml`/`wind_IndicatorTree.dat` 加密仅 DLL 可解）。故 Wind 改用**方案 C 探测法**：以常用字段候选清单逐字段调 `/wind/wss` 实测，保留 Wind 确认存在的 130 个常用字段。详见 [`.claude/plans/wind-catalog-crawl.plan.md`](../.claude/plans/wind-catalog-crawl.plan.md)。

### Track A 的边界（为什么不是全集）
- EM `sector` 返回的是**板块成分**（股票代码），不是指标清单。
- EM `datastatistics` 返回的是**函数流量/额度**，`FUNCENAME` 是函数名（EM_CSS…），不是指标。
- iFinD `THS_DataPool` / `THS_DataStatistics` 同理：枚举成分与流量，非指标字典。
- 两个 SDK 安装目录里**没有**任何随包字典文件（xls/csv/json）。

→ 所以财务全量指标代码、技术指标全集**走命令生成器后台接口**(EM=Track C 已完成，iFinD=Track C-iFinD 已完成)。
EDB 宏观指标树两家均暂跳(数万条，按需定点拉)。

## Track C-iFinD（同花顺全量，2026-06-29 完成）

iFinD 与 EM 同法,走 `quantapi.51ifind.com` 命令生成器(超级命令)后台接口。**关键踩坑**:
- **不是 IP 封**:同 IP 用 curl 打登录端点正常,只有自动化浏览器(CDP)被反爬拦;普通浏览器正常登录。
- **session 绑登录浏览器**:cookie 搬到服务器 curl 报 `-1010 已登出`,故抓取必须在用户已登录浏览器的 Console 就地 fetch。
- 接口:`get_api_function_tree`(函数→品种)、`list_by_seq?seq=&type=`(按品种递归指标树)、`block_from_config`(板块树)。
- 脚本:`scripts/catalog/ifind_crawl.js`(浏览器爬虫)→ `parse_ifind.py`(JSON→CSV)→ `gen_md_ifind.py`(CSV→手册)。
- 原始 JSON(70MB)存 `docs/catalog/_raw/ifind_catalog.json`(已 gitignore)。详见记忆 `ifind-catalog-webapi-breakthrough`。
