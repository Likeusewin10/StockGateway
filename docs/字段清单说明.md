# 两家系统字段级指标清单 — 说明与覆盖度

> 面向交易系统设计：盘清东方财富 Choice / 同花顺 iFinD 两家的**指标体系**，
> 标注哪些已由 API 程序化拿到（Track A），哪些必须用桌面端超级命令导出（Track B）。

## 结论（先看这条）

**东方财富 EM/Choice 的全量指标字典已拿到**（截面 11823 + 序列 570 + 专题报表字段 6119），
不是靠桌面客户端 GUI，而是靠**官网"命令生成器"的后台 JSON 接口**（quantapi.eastmoney.com，网页登录即可，
绕开了桌面端"导出权限"墙）。

- 桌面客户端的"数据浏览器/超级命令"测试账号**无导出权限**，此路不通。
- 改走**官网命令生成器**（产品手册→命令生成）：网页用 Choice/东财账密登录，
  其分类树接口 `GetIndicatorCategoryInfos` + 叶子明细接口 `GetLeafNodeOnIndicatorCategoryInfoBy`
  直接返回每个指标的代码(EngName)/中文名/单位/参数/适用范围/释义/算法。

### Track A 的边界（为什么不是全集）
- EM `sector` 返回的是**板块成分**（股票代码），不是指标清单。
- EM `datastatistics` 返回的是**函数流量/额度**，`FUNCENAME` 是函数名（EM_CSS…），不是指标。
- iFinD `THS_DataPool` / `THS_DataStatistics` 同理：枚举成分与流量，非指标字典。
- 两个 SDK 安装目录里**没有**任何随包字典文件（xls/csv/json）。

→ 所以财务全量指标代码、技术指标全集、EDB 指标树**只能走 Track B**。

## Track B 操作指引（你装好客户端后按此导出）

详细分步见 `docs/超级命令导出指引.md`。一句话版：

1. **Choice 数据终端**：顶部「数据浏览器 / EDB」→ 按维度（行情/财务/估值/技术/宏观）选指标 → 右键/工具栏「导出 Excel」。
2. **iFinD 终端**：「数据接口 / 超级命令」→ 指标库按维度展开 → 「导出」。
3. 导出文件放 `docs/catalog/_raw/`，我来归一化成统一 schema 合并进 `docs/catalog/`。
