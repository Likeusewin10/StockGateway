# 远程调用接口文档

供 Mac / 其他项目跨网络调用本股票数据服务。调用方**不需要安装任何股票 SDK、也不需要装任何客户端**，
只要发 HTTP GET 请求 + 带密钥鉴权头，即可通过固定公网域名取数。

## 一、连接信息

| 项 | 值 | 说明 |
|---|---|---|
| 服务器地址 | `https://jtx.ngrok.app` | 固定公网域名，HTTPS；任意设备可达 |
| 鉴权头 | `X-API-Key: USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y` | **所有取数接口必填**，缺失/错误一律 401 |
| 交互式文档 | `https://jtx.ngrok.app/docs` | 浏览器打开可直接点着试 |
| 探活 | `GET /health` | 不需要鉴权，返回 `{"status":"ok"}` |

> 域名固定不变。隧道由计划任务 `NgrokTunnel` 托管（登录自启 + 每分钟自愈），与数据服务同款常驻机制。
> 链路经公网节点中转，单次请求约 1.5~2.5 秒；偶发瞬时断连，重试即可。

## 二、鉴权（必读）

**除 `/health` 外的所有取数接口都强制要求密钥鉴权**：请求头必须带 `X-API-Key`，
且其值与服务器 `.env` 里的 `API_KEY` 完全一致，否则一律返回 `401 Unauthorized`，取不到任何数据。

- 当前 key：`USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y`
- key 不要发到公开渠道（聊天群、issue、前端代码）；如泄露，在服务器 `.env` 改 `API_KEY` 后重启 `StockDataService` 即可全量失效旧 key。

```bash
# 不带 key 或 key 错误 → 401；带正确 key → 200
curl -H "X-API-Key: USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y" \
  "https://jtx.ngrok.app/em/csd?codes=300059.SZ&indicators=CLOSE&startdate=2024-01-02&enddate=2024-01-03"
```

## 三、接口一览

所有接口均为 **GET**，参数走查询字符串，返回 JSON。

| 接口 | 说明 | 必填参数 | 可选参数 |
|---|---|---|---|
| `/health` | 健康检查（免鉴权） | 无 | 无 |
| `/em/csd` | 东财序列数据 | `codes` `indicators` `startdate` `enddate` | `options` |
| `/em/css` | 东财截面数据 | `codes` `indicators` | `options`（如 `TradeDate=20240105`） |
| `/ths/history` | 同花顺历史行情 | `codes` `indicators` `begin` `end` | `params` |
| `/ths/basic` | 同花顺基础/截面 | `codes` `indicators` | `params` |
| `/ths/realtime` | 同花顺实时行情 | `codes` `indicators` | `params` |

参数约定：
- **EM（东财）**：`codes` / `indicators` 多个用**逗号**分隔，日期格式 `YYYY-MM-DD`。
- **iFinD（同花顺）**：`codes` 多个用**逗号**，`indicators` 多个用**分号**，日期格式 `YYYY-MM-DD`。

### 通用透传接口（覆盖两个 SDK 的全部方法）

上面 5 个是常用的便捷封装；**其余所有 SDK 方法**（EDB 宏观、专题报表 `ctr`、条件选股 `cps`、板块 `sector`、交易日历、智能问财 `iwencai`、数据池 `DataPool`、财报 `ReportQuery` 等）无需逐个加端点，统一走两个**通用透传**接口，用 **POST** 传参：

| 接口 | 方法 | 说明 |
|---|---|---|
| `GET /em/methods` | GET | 列出东财 `c` 的所有方法及类别标注（均可调用，状态以 `ok` 开头） |
| `POST /em/call/{method}` | POST | 调用 `c.<method>(*args)`，body：`{"args":[...], "options_pandas":true}` |
| `GET /ths/funcs` | GET | 列出 iFinD 所有 `THS_` 函数及类别标注（均可调用，状态以 `ok` 开头） |
| `POST /ths/call/{func}` | POST | 调用 `THS_<func>(*args, **kwargs)`，body：`{"args":[...], "kwargs":{...}}` |

- `options_pandas`（仅 EM）默认 `true`：自动给最后一个 options 串补 `Ispandas=1`，返回转成 JSON 记录数组。
- **全部方法均可调用，不再拦截**。`/em/methods`、`/ths/funcs` 会给出类别标注（均以 `ok` 开头），方便你判断用法：
  - `ok`：常规取数。
  - `ok(交易/组合)`：东财组合管理类（`pcreate/porder/pquery/preport/pctransfer/pdelete`），操作虚拟组合持仓，无真实资金流出。
  - `ok(异步...)`：异步推送类（EM `csq/chq/cst/cnq`、iFinD `THS_Asy*`）。回调模型不适配同步 HTTP，**请改用 WebSocket**（见下文），直接同步调用拿不到推送结果。
  - `ok(会话管理,谨慎...)`：⚠️ 会改服务自身的 SDK 会话状态，**会影响整个服务的所有调用方**。例如 `POST /em/call/stop` 会登出整个服务的东财会话、`THS_iFinDLogout` 会登出同花顺会话，导致其他人取数失败直到服务重启自愈。除非你清楚后果，否则不要调用这类方法。
- 先调 `/em/methods`、`/ths/funcs` 看清方法名怎么拼、属于哪类。

```bash
# 东财 EDB 宏观数据（原来没有专门端点，走透传）
curl -X POST -H "X-API-Key: USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y" \
  -H "Content-Type: application/json" \
  -d '{"args":["EMM00087117","IsPublishDate=1,RowIndex=1"]}' \
  "https://jtx.ngrok.app/em/call/edb"

# 东财组合下单（虚拟组合持仓加减，无真实资金）：porder(combincode, orderdict, remark)
# orderdict: code/volume/price/date 列表对齐；optype 1买入 2卖出
curl -X POST -H "X-API-Key: USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y" \
  -H "Content-Type: application/json" \
  -d '{"args":["mypf.PF",{"code":["300059.SZ"],"volume":[1000],"price":[13.11],"date":["2024-01-05"],"optype":[1]},"备注"],"options_pandas":false}' \
  "https://jtx.ngrok.app/em/call/porder"

# 同花顺 EDB（位置参数顺序同 SDK：indicators, param, begin, end）
curl -X POST -H "X-API-Key: USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y" \
  -H "Content-Type: application/json" \
  -d '{"args":["M001620326","","2024-01-01","2024-06-01"]}' \
  "https://jtx.ngrok.app/ths/call/EDB"
```

### WebSocket 实时推送（真·订阅推送）

需要持续盯盘、不想轮询时，用 WebSocket 长连接。SDK 行情回调一触发就顺着连接推下来。

| 端点 | 说明 |
|---|---|
| `WS /em/ws?key=<API_KEY>` | 东财实时行情推送（底层 `csq`） |
| `WS /ths/ws?key=<API_KEY>` | 同花顺实时行情推送（底层 `QuotesPushing`） |

- **鉴权走 query 参数** `?key=`（WebSocket 不便带请求头），值同 `API_KEY`；错误则返回 error 帧并断开。
- 连接后发 JSON 文本帧订阅，可多次订阅多个标的；**断开连接时服务端自动退订全部**。
- 协议（客户端 → 服务端）：
  - 订阅：`{"action":"subscribe","codes":"300059.SZ","indicators":"now,open"}`
  - 退订：EM `{"action":"unsubscribe","serial":<订阅号>}`；iFinD `{"action":"unsubscribe","codes":"300033.SZ","indicators":"latest"}`
- 服务端 → 客户端帧：`{"event":"subscribed",...}` / `{"event":"quote","data":...}` / `{"event":"error","msg":...}`
- ⚠️ **依赖账号实时推送权限**：当前东财账号 `csq` 无权限（订阅返回 `10001012 insufficient user access`）；同花顺可订阅成功。**非交易时段无 tick 推送属正常**。
- ⚠️ 公网经 ngrok 的长连接偶发断开，客户端需自带重连逻辑。

```python
# pip install websockets
import asyncio, json, websockets

async def main():
    url = "wss://jtx.ngrok.app/ths/ws?key=USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y"
    async with websockets.connect(url) as ws:
        await ws.send(json.dumps({"action":"subscribe","codes":"300033.SZ","indicators":"latest"}))
        async for msg in ws:
            print(json.loads(msg))

asyncio.run(main())
```

## 五、在代码里调用（任意语言；以 Python 为例）

消费端**不依赖任何股票 SDK**，只用普通 HTTP 客户端：

```python
import requests

BASE = "https://jtx.ngrok.app"
HEADERS = {"X-API-Key": "USp6FKBUuL0dbCJQNNz3sl6hKhmJqE6y"}  # 鉴权，必填

r = requests.get(f"{BASE}/em/csd", headers=HEADERS, params={
    "codes": "300059.SZ", "indicators": "CLOSE",
    "startdate": "2024-01-02", "enddate": "2024-01-05",
}, timeout=30)
r.raise_for_status()
print(r.json())
# [{"CODES":"300059.SZ","DATES":"2024/01/02","CLOSE":13.8}, ...]
```

> 建议设 `timeout`：公网中转时延偏高（1~3 秒），别用过短的超时。

## 六、返回与错误码

| 状态码 | 含义 | 处理 |
|---|---|---|
| `200` | 成功，body 为 JSON 数据 | 正常解析 |
| `400` | 参数与 SDK 方法签名不匹配（透传时常见） | 对照 `/em/methods`、`/ths/funcs` 和 SDK 文档核对 args |
| `401` | 缺少或错误的 `X-API-Key` | 检查请求头是否带、key 是否与服务器 `.env` 一致 |
| `403` | 调用了私有方法（名字以 `_` 开头） | 只调公开方法 |
| `404` | 方法名拼错 / 不存在 | 用 `/em/methods`、`/ths/funcs` 确认方法名 |
| `429` | 触发限流（每 IP 每 60 秒最多 60 次取数请求） | 降低请求频率，稍后重试；高频批量取数请合并参数 |
| `502` | SDK 登录失败 / 取数无权限 / 流量用尽 | 看 body 里的原始错误码，多为账号侧问题 |
| 连接超时 | 服务/隧道没起 或 网络不通 | 见下方排查 |

> **限流**：服务对取数接口（`/em/*`、`/ths/*`，含透传）按来源 IP 限流，**每 60 秒最多 60 次**，超出返回 `429`。`/health` 不限流。限流计数在服务进程内，重启即重置。
> **CORS（浏览器跨域）**：默认**拒绝**所有跨域请求。如需从浏览器前端直连，需在服务器 `.env` 的 `CORS_ORIGINS` 把你的站点域名加入白名单（逗号分隔）；服务端调用（curl/requests/后端）不受此限制。

## 七、连不上时的排查（按顺序）

1. **先探活**：`curl https://jtx.ngrok.app/health`。通 → 服务正常，问题在你的请求（多半是 key）。不通 → 往下查。
2. **瞬时断连**：公网节点偶发抖动，先原样重试 1~2 次。
3. **服务层**（需到服务器上）：两套计划任务托管，均登录自启 + 每分钟自愈：
   - 数据服务 `StockDataService`：`schtasks /Run /TN StockDataService`；状态 `Get-ScheduledTask -TaskName StockDataService | Get-ScheduledTaskInfo`。
   - 隧道 `NgrokTunnel`：`schtasks /Run /TN NgrokTunnel`；日志在项目目录 `ngrok.log`。

## 八、注意事项

- **限频/流量**：EM 截面/序列每分钟 ≤700 次，并按品种有流量额度；iFinD 为单点登录、且服务内部用全局锁串行化所有请求，高并发会排队变慢。
- **单点登录**：iFinD 账号同一时间只能一处登录，本服务占用该登录位；别处再登会互相挤掉。
- **安全**：服务暴露在公网，**唯一防线是 API Key**。务必保管好 key，不要发到公开渠道；怀疑泄露立即在服务器 `.env` 改 `API_KEY` 并重启 `StockDataService`。
